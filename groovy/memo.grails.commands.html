執行grail test-app會先跑unit test之後接著跑integration test。如果想要只跑unit test或integration test可以在指令後面加-unit或-integration讓grails執行unit test或integration test。

create-script

建立一個可以在terminal透過grails執行的script，產生的script會放在scripts目錄下。

schema-export

export DDL to file or console，當然必須要有正確的hibernate dialect設定。

set-version

設定project root directory的application.properties (Grails官方文件稱這個檔案為application metadata file，在GSP裡面可以透過<g:meta name="property_key" />這個tag取得檔案裡面的資訊)裡的app.version的value。

Static Scaffold

產生static scaffold有下列指令

grails generate-all grails generate-controller grails generate-views

如果覺得開發的project要用特定的scaffold template來generate的話，可以使用下列指令把那些template安裝到project裡面

grails install-templates

安裝的檔案在/src/templates下。/src/templates/artifacts裡放的是create-xxxxxx用的template、/src/templates/scaffolding是scaffold使用的template，最後/src/templates/war放的是web.xml的template。
